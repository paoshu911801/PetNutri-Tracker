針對新增資料庫
輸入數據需有每100克的各數據
粗蛋白、粗纖維、粗脂肪為％，當使用者輸入％，後台計算 * 100得出公克
若是使用者輸入單位為mg/kg，後台計算需 / 1000，得出公克

針對每日紀錄
輸入數據為公克
後台資料庫數據為100克
所以當輸入數據後，後台所有營養數據計算法為 (輸入數據 / 100) * 營養數據

針對週報（分級）
以每周總分數分級
營養守護神(90up)｜營養守門員(between40~90)｜營養同伴(under40)

針對週報（分數配比）
資料輸入完整度 = {(本週輸入天數 * 欄位) / [每日紀錄共3欄位(食物、吃完與否、大便) * 7(一週7天)]} * 100
對資料庫的貢獻 = 本週有資料輸入資料庫，一次得5分
本週登入天數 = (登入天數 / 7) * 100
週報呈現的成績 = (資料輸入完整度 + 本週登入天數) / 2 + 對資料庫的貢獻

針對週報（食慾分析）
常吃不完 = (總沒吃完餐數 / 一週餐數) * 100，>50
每次吃完 = [1 - (總沒吃完餐數 / 一週餐數)] * 100，>50

針對週報（營養分析）
過剩：(熱量吃太多 / 7(一週)) * 100，>70
不足：(熱量吃太多 / 7(一週)) * 100，<40
完美：(熱量吃太多 / 7(一週)) * 100，介於40~70

針對週報（腸胃分析）
便便正常：(正常便便 / 總便便次數) * 100，>80
過軟：(軟便便 / 總便便次數) * 100，>60

針對週報（建議攝取）
每日攝取缺乏營養素統計，提供建議，搜尋線上最符合補充的營養補充品，給予建議

針對週報（健康評估）
常吃不完、營養不足、軟便，3取2則顯示獸醫建議
